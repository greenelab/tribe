Servers connected to Tribe allow users to put their collections to work within external servers.
GIANT's OAuth2 connection to Tribe allows users to interpret their own collections through tissue-specific networks. For example, user-owned collections as well as publicly available collections are used for biological process enrichment calculations (shown above). Because GIANT is connected to the Tribe web service, GIANT's users benefit from up to date process annotations and can use GIANT to study processes not covered by curated databases but included in their own Tribe collections.
Link your server to Tribe through OAuth2 to allow users to access their own non-public collections within your server.
Connecting to Tribe with an example server (e.g. GIANT). The user visits GIANT and indicates that they'd like to analyze a non-public Tribe collection (1). GIANT redirects the user to a Tribe login page with a token indicating that the request came from GIANT (2). The user logs into Tribe (3) and Tribe returns an authorization code to the browser (4) which is automatically sent to GIANT (5). GIANT then sends the authorization code to Tribe (6) which returns an access token (7) that allows the server to retrieve that user's private collections through the standard Tribe API. The server then returns the desired results to the user (8). This process allows a user to analyze private collections on external servers while keeping their login information private. We provide an open source Tribe client written in python that performs these tasks.
